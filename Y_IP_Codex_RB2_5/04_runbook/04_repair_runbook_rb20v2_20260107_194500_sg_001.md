### 1.2 计算需要修复的 shard 列表（必须执行，并与“期望列表”对齐）

定义：出现以下任一情况即需修复
- `source_members_cnt>0 AND map_member_block_natural_cnt=0`（自然块映射缺失）
- `preh_blocks_cnt>0 AND (block_final_cnt=0 OR map_member_block_final_cnt=0 OR profile_final_cnt=0)`（PreH 存在但 Final 链路缺失）

```sql
WITH sp AS (
  SELECT shard_id FROM rb20_v2_5.shard_plan WHERE run_id='rb20v2_20260107_194500_sg_001'
),
sm AS (
  SELECT shard_id, COUNT(*)::bigint AS cnt
  FROM rb20_v2_5.source_members
  WHERE run_id='rb20v2_20260107_194500_sg_001'
  GROUP BY 1
),
mapn AS (
  SELECT shard_id, COUNT(*)::bigint AS cnt
  FROM rb20_v2_5.map_member_block_natural
  WHERE run_id='rb20v2_20260107_194500_sg_001'
  GROUP BY 1
),
preh AS (
  SELECT shard_id, COUNT(*)::bigint AS cnt
  FROM rb20_v2_5.preh_blocks
  WHERE run_id='rb20v2_20260107_194500_sg_001'
  GROUP BY 1
),
bf AS (
  SELECT shard_id, COUNT(*)::bigint AS cnt
  FROM rb20_v2_5.block_final
  WHERE run_id='rb20v2_20260107_194500_sg_001'
  GROUP BY 1
),
mapf AS (
  SELECT shard_id, COUNT(*)::bigint AS cnt
  FROM rb20_v2_5.map_member_block_final
  WHERE run_id='rb20v2_20260107_194500_sg_001'
  GROUP BY 1
),
pf AS (
  SELECT shard_id, COUNT(*)::bigint AS cnt
  FROM rb20_v2_5.profile_final
  WHERE run_id='rb20v2_20260107_194500_sg_001'
  GROUP BY 1
)
SELECT
  sp.shard_id,
  COALESCE(sm.cnt,0)   AS source_members_cnt,
  COALESCE(mapn.cnt,0) AS map_natural_cnt,
  COALESCE(preh.cnt,0) AS preh_blocks_cnt,
  COALESCE(bf.cnt,0)   AS block_final_cnt,
  COALESCE(mapf.cnt,0) AS map_final_cnt,
  COALESCE(pf.cnt,0)   AS profile_final_cnt
FROM sp
LEFT JOIN sm   USING (shard_id)
LEFT JOIN mapn USING (shard_id)
LEFT JOIN preh USING (shard_id)
LEFT JOIN bf   USING (shard_id)
LEFT JOIN mapf USING (shard_id)
LEFT JOIN pf   USING (shard_id)
WHERE
  (COALESCE(sm.cnt,0) > 0 AND COALESCE(mapn.cnt,0) = 0)
  OR
  (COALESCE(preh.cnt,0) > 0 AND (COALESCE(bf.cnt,0) = 0 OR COALESCE(mapf.cnt,0) = 0 OR COALESCE(pf.cnt,0) = 0))
ORDER BY sp.shard_id;
```

把上述查询返回的 `shard_id` 集合记为 **REPAIR_SHARDS**，它是本次修复的唯一准入集合（后续所有 `IN (...)` 都必须使用它）。  

当前我最新核验到的 REPAIR_SHARDS 为：`1,3,5,9,11,13,20,27,51,56`  
若你查询出来与此不同：**不要自行猜测**，把你查到的列表发我确认；但如果你能明确它仍满足“定义条件”（source>0 且 map_natural=0；或 preh>0 且 final 链路缺失），你也可以直接把该列表作为 REPAIR_SHARDS 继续执行（并在回传信息里说明差异）。

sm AS (SELECT shard_id, count(*)::bigint AS cnt FROM rb20_v2_5.source_members WHERE run_id='rb20v2_20260107_194500_sg_001' GROUP BY 1),

WITH sp AS (SELECT shard_id FROM rb20_v2_5.shard_plan WHERE run_id='rb20v2_20260107_194500_sg_001'),
preh AS (SELECT shard_id, count(*)::bigint AS cnt FROM rb20_v2_5.preh_blocks WHERE run_id='rb20v2_20260107_194500_sg_001' GROUP BY 1),
