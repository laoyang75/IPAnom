# DP-001：中国过滤谓词（字段：`IP归属国家`）

决策：按你的“其他都按推荐”已确认 **选A**（`IP归属国家 IN ('中国')`）。

背景（主版本要求）：RB20_01 必须过滤非中国；但“哪些取值算中国”不在主版本写死，必须先采样 distinct 值交人类确认，再固化到 Config 合同。

MCP 实测证据（W 源表）：
- `IP归属国家` distinct = 1，且唯一取值为 `中国`（全量 59,706,088 行）

你只需回复：`选A` / `选B` / `选C`

## 选项

### A（推荐）
采样 distinct 后，采用白名单集合：
- 谓词：`IP归属国家 IN (<确认值集合>)`
  - 建议本次集合直接为：`('中国')`

### B
采样 distinct 后，采用你给出的自定义布尔表达式（SQL WHERE 条件）：
- 例：`IP归属国家 ILIKE '%中国%'`（仅示例，最终以你给出的表达式为准）

### C
采样 distinct 后，维护“国家取值→是否中国”的配置表（仍需合同写死表名、主键与 join 口径）：
- 谓词：join 配置表后取 `is_cn=true`

## 固化位置（被选中后）

- `02_contracts/config_contract.*`（或写入 `02_contracts/schema_contract.*` 的 Config 表定义）
