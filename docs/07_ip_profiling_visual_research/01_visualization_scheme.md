## 2. Layer 1: 库总览 Dashboard

### 2.1 三库卡片

每个库（H/E/F）一张卡片，展示：

| 指标 | H 库 | E 库 | F 库 |
|------|------|------|------|
| 成员总量 | 13M | 45M | 1.7M |
| 有效率 | 有效/总量 | 有效/总量 | 有效/总量 |
| 运营商集中度 | TOP3 运营商占比 | TOP3 占比 | TOP3 占比 |
| 上报次数中位数 | median(上报次数) | median | median |
| 移动设备比例 | mobile_ratio | ratio | ratio |
| 问题数据量 | 异常标记数 | 异常标记数 | 异常标记数 |

### 2.2 流向 Sankey 图

```
Source Members (59.7M)
  ├── 异常标记 → 标记但保留
  ├── Drop (ALL_ABNORMAL_BLOCK) → 丢弃
  └── Keep
       ├── Phase 03 网络规模评分
       │    ├── 中型网络+ → H 库准入
       │    └── 其他 → R1 残余
       ├── Phase 07 /27 原子密度
       │    ├── 密度≥0.2 + run≥3 → E 库
       │    └── 未达标 → 继续
       └── Phase 08 最终筛选
            └── R1 \ E → F 库
```

### 2.3 数据源 SQL

```sql
-- 库级统计 (Layer 1 核心查询)
SELECT 
  'H' AS library,
  COUNT(*)::bigint AS member_cnt,
  COUNT(*) FILTER(WHERE sm.is_valid)::bigint AS valid_cnt,
  COUNT(*) FILTER(WHERE sm.is_abnormal)::bigint AS abnormal_cnt,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY sm."上报次数") AS median_report,
  AVG(CASE WHEN sm."设备数量" > 0 
       THEN sm."移动网络设备数量"::float / sm."设备数量" 
       ELSE 0 END) AS avg_mobile_ratio
FROM rb20_v2_5.h_members h
JOIN rb20_v2_5.source_members sm USING (run_id, ip_long)
WHERE h.run_id = :run_id

UNION ALL
-- 同样的查询 for E, F...
```

## 3. Layer 2: 单库深度画像

### 3.1 处理流程 Timeline

以 **H 库** 为例，展示从 Source 到最终进入 H 库的全过程：

```
Phase 01: source_members
├── 输入: public.ip源表 中的中国 IP
├── 处理: 异常标记（LEFT JOIN 异常表）
├── 输出: 59.7M 行
└── ⬇️ 进入 H 的子集: 全部

Phase 02: block_natural
├── 输入: source_members (59.7M)
├── 处理: 连续 ip_long 聚合成自然块
├── 输出: ~13.3M 自然块
└── 可视化: 块大小分布直方图（最终进入H的块 vs 不进入H的块）

Phase 03: profile_pre
├── 输入: block_natural
├── 处理: SIMPLE 评分 → network_tier
├── 输出: 预画像
├── 关键变化: Keep/Drop 决策
│   └── 可视化: Score 分布图 + 阈值线标注
└── ⬇️ 进入 H 的子集: keep=true 的块

Phase 04: split → block_final
├── 输入: 窗口统计 + 预画像
├── 处理: 三触发器切分
├── 输出: 最终块
├── 关键变化: 块被切分后 network_tier 可能降级
│   └── 可视化: 切分前后 tier 变化 Sankey
└── ⬇️ 进入 H 的子集: tier_final = '中型网络'

Phase 05: h_blocks + h_members
├── 准入条件: network_tier_final = '中型网络'
├── 输出: H 库最终成员 ~13M
└── 可视化: 准入/拒绝比率 + 拒绝原因分布
```

### 3.2 特征分布图表

针对每个库，按以下维度生成分布图：

| 图表类型 | X 轴 | Y 轴 | 说明 |
|----------|------|------|------|
| 直方图 | density | 块数量 | 密度分布，标注阈值线 |
| 直方图 | valid_cnt | 块数量 | 有效成员数分布 |
| 散点图 | density | valid_cnt | 二维特征空间，颜色=network_tier |
| 饼图 | network_tier | 占比 | 网络规模层级分布 |
| 箱线图 | 运营商 | 上报次数 | 各运营商上报行为差异 |
| 时间序列 | 处理阶段 | 成员数 | 各阶段漏斗变化 |

### 3.3 问题数据面板

```
问题类型分类：
├── 1. 阈值边界问题
│   └── 哪些 IP 的 score/density 恰好在阈值附近？稳定性如何？
├── 2. 异常聚集问题
│   └── 某些块的异常率明显高于同库平均？为什么？
├── 3. 分类一致性问题
│   └── 相似特征的 IP 是否被分到了不同的库？
├── 4. 数据质量问题
│   └── NULL 值、极端值、不合理的上报次数/设备数量
└── 5. 切分合理性问题
    └── Phase 04 切分是否将本应在一起的 IP 拆开？
```

## 4. Layer 3: Agent 交互面板

### 4.1 操作界面设计

```
┌─────────────────────────────────────────────────┐
│               Agent 研究助手                     │
│  ┌──────────────────────────────────────────┐   │
│  │ 📊 当前研究上下文                         │   │
│  │ 库: H | 阶段: Phase 03 | 焦点: 阈值边界  │   │
│  └──────────────────────────────────────────┘   │
│                                                  │
│  💬 对话区                                       │
│  ┌──────────────────────────────────────────┐   │
│  │ [User] score=19 和 score=20 的块差异?     │   │
│  │ [Agent] 分析如下:                         │   │
│  │   - score=19 块: 3,245 个, 均在"小型网络" │   │
│  │   - score=20 块: 2,189 个, 均在"中型网络" │   │
│  │   - 影响成员数差异: 42万                   │   │
│  │   建议: 检查 wD=4→16 跳跃的影响范围       │   │
│  └──────────────────────────────────────────┘   │
│                                                  │
│  📋 操作日志                                     │
│  ┌──────────────────────────────────────────┐   │
│  │ 12:30 查看 H 库 density 分布              │   │
│  │ 12:35 发现 score=19/20 边界异常           │   │
│  │ 12:40 Agent 分析: wD 阈值造成 42万成员... │   │
│  │ 12:45 决策: 增加 wD 中间档位 (30~100→8)   │   │
│  │ 12:50 Agent 执行: 模拟新阈值影响          │   │
│  └──────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

### 4.2 Agent 能力清单

| 能力 | 说明 |
|------|------|
| **统计查询** | 按任意维度聚合查询库内数据 |
| **对比分析** | 对比不同库、不同阶段、不同阈值的数据差异 |
| **模拟评估** | 模拟参数调整后的分类结果变化 |
| **SQL 生成** | 生成修正 SQL 并解释逻辑 |
| **日志记录** | 自动记录每步操作和决策理由 |
| **报告生成** | 将研究过程整理为可复用的标准化规则文档 |

```yaml

library: H

  - rule: "wD 分桶调整"
    before: "30~200 → 16"
    after: "30~100 → 8, 100~200 → 16"
    reason: "减少阈值边界的分类跳跃"

  - rule: "自然块 gap 容差"
```
