# 合理性分析与问题审计

> 对 RB20 v2.5 Pipeline 的核心算法、数据结构和业务规则进行合理性审计。

## 1. 守恒性分析

### 1.1 Keep = H ∪ E ∪ F 守恒

**数学证明路径**:

```
KeepMembers = Source \ Drop
H_cov = {m ∈ Keep | m 归属的 block_final.network_tier_final = '中型网络'}
R1 = Keep \ H_cov
E_cov = {m ∈ R1 | m 归属的 atom27 满足密度准入 ∧ 所属 run_len ≥ 3}
F = R1 \ E_cov (通过 atom27 等值 anti-join)

∴ Keep = H_cov ∪ R1
       = H_cov ∪ (E_cov ∪ F)
       = H_cov ∪ E_cov ∪ F
```

**✅ 合理**: 守恒性由集合代数保证，且 Phase 99 `conservation_keep_equals_hef` 断言实施了数据级验证。

**🟡 潜在风险**: 守恒依赖 `atom27 等值 anti-join` 精确实现。如果 F 的构造使用了 `BETWEEN` 或范围 anti-join，可能因"幽灵成员"破坏守恒。当前 `重构2.md` 已明确 **禁止 BETWEEN**。

---

### 1.2 互斥性 H∩E=∅, H∩F=∅, E∩F=∅

**逻辑保证**:
- H 从 Keep 中按 block_final 分类提取
- R1 = Keep \ H（集合差运算，天然排除 H）
- E 从 R1 中提取 → E∩H = ∅（前提保证）
- F = R1 \ E → F∩E = ∅（集合差运算）
- F ⊆ R1 且 R1∩H = ∅ → F∩H = ∅

**✅ 合理**: 互斥由构造顺序保证（H先行、R1排除H、E/F从R1互斥提取）。

## 2. Natural Block 定义合理性

### 2.1 连续区间 vs 传统 CIDR

| 维度 | 传统 CIDR 前缀块 | 本项目自然块 |
|------|-----------------|-------------|
| 定义 | 按 /n 前缀对齐 | `ip_long` 连续差=1 |
| 对齐要求 | 必须 2^n 对齐 | 无对齐要求 |
| 灵活性 | 低（固定大小） | 高（自适应实际分布）|
| 与 ISP 分配对应 | 较好 | 较差 |

**✅ 合理**: 本项目的核心目标不是还原 ISP 路由分配，而是基于**实际 IP 成员分布**进行聚类分析。连续区间定义更贴合实际数据的物理分布特征，避免了 CIDR 前缀对齐导致的块分裂或块合并误差。

**🟡 需注意**: 自然块的大小高度依赖于源表中 IP 的连续性。如果源表缺失中间 IP（例如非中国 IP 被过滤后留下间隙），可能导致大块被人为切断。

**建议**: 未来可考虑"允许 gap=1~3 的容差合并"以减少因单个 IP 缺失导致的块碎片化。需评估对下游切分和分类的影响。

## 3. SIMPLE 评分函数审计

### 3.1 wA 分桶 (valid_cnt)

```
已知数据: 微型网络 13.1M 块, 小型网络 1.5万块, 中型网络 1.4万块,
          大型网络 1.2万块, 超大网络 584块
```

| 区间 | wA | 评估 |
|------|----|------|
| 1~16 | 1 | ✅ 合理：极小块给最低权重 |
| 17~48 | 2 | ✅ 合理：小规模成员 |
| 49~128 | 4 | ✅ 合理：中等规模 |
| 129~512 | 8 | ✅ 合理：大规模成员 |
| ≥513 | 16 | 🟡 注意：权重跳跃大（8→16），极大块可能与 512 块的差异被过度放大 |

**分析**: wA 权重分桶使用了指数增长模式（1→2→4→8→16），这在实践中常用但存在 **阈值敏感性**——512 和 513 成员的块，wA 从 8 跳到 16，simple_score 直接增加 8 分，可能导致 network_tier 跨级。

**建议**: 如线上化后需要精细控制，可考虑改用连续函数（如 `log2(valid_cnt)` 加权），但当前离散分桶在批处理场景下足够实用。

### 3.2 wD 分桶 (density)

| 区间 | wD | 评估 |
|------|----|------|
| ≤3.5 | 1 | ✅ 低密度 |
| 3.5~6.5 | 2 | ✅ 中低密度 |
| 6.5~30 | 4 | 🟡 区间跨度大（6.5→30，4.6倍），可能掩盖差异 |
| 30~200 | 16 | 🟡 权重跳跃大（4→16），且区间跨度也大 |
| >200 | 32 | ✅ 超高密度极端值 |

**关键问题**: wD 在 4→16 之间存在 **12 分的跳跃**，这意味着 density 从 30 到 30.1 的微小变化会导致 simple_score 增加 12 分，足以从"小型网络"直接跳到"大型网络"。

**建议**: 考虑增加一个中间档位（如 30~100 → 8, 100~200 → 16），平滑 wD 的阶梯效应。

### 3.3 network_tier 阈值

**simple_score 范围分析**:
- 最低: wA=1 + wD=1 = 2（微型网络 ✅）
- 中间: wA=4 + wD=4 = 8（微型网络）→ wA=8 + wD=2 = 10（小型网络 ✅）
- 中型准入: simple_score ≥ 20，需要 wA + wD ≥ 20
  - 例: wA=4(49~128) + wD=16(30~200) = 20 ✅
  - 或: wA=16(≥513) + wD=4(6.5~30) = 20 ✅

**✅ 合理**: 中型网络（H类准入）要求"要么成员多且密度高，要么成员极多"，符合业务意图。

## 4. /27 原子密度阈值审计

- 阈值: `atom_density ≥ 0.2` 即 `valid_cnt ≥ 7`（32 个 IP 中至少 7 个有效）
- 连续 run 最小长度: 3 个原子

**数据验证**:
- E 成员 ~45M，占 Keep 的 75.3%
- 如果阈值过低（如 0.1 即 ≥4），E 会吞噬更多 F
- 如果阈值过高（如 0.3 即 ≥10），E 减少，F 膨胀

**✅ 合理**: 0.2 产生了 75% E、22% H、3% F 的分布，H 和 E 覆盖了 97% 的 Keep 成员，F 仅为残余散点，符合"IP 地址大多集中在密集网段"的现实分布特征。
