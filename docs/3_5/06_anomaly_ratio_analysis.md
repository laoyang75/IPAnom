## 2. 适用边界与特殊值

### 2.1 公式不适用的场景

| 场景 | 特征 | 原因 | 处理方式 |
|------|------|------|----------|
| **单 IP 块** | ip_count = 1 | 无法做连续性分析，所有指标退化为单点值 | **单独标签**：不用比例公式，用绝对值判断 |
| **云服务器出口** | 运营商=阿里云/腾讯云/华为云 | 代理/VPN 出口，app 极少但设备极多 | **专用标签**：云出口（operator_is_cloud） |
| **超小块** | ip_count ≤ 3 且 dev/IP ≤ 5 | 样本量太小，比例不稳定，随机性大 | **豁免**：不参与比例检测，仅用绝对值 |
| **超大 NAT** | dev/IP > 10,000 | 省级 NAT 出口，维度饱和 | **特殊处理**：用 app/device < 0.03 检测 |

### 2.2 适用范围

```
比例公式适用条件:
  ip_count ≥ 4
  AND avg_devices_per_ip ≥ 5
  AND avg_devices_per_ip ≤ 10000
  AND top_operator NOT IN ('阿里云','腾讯云','华为云','亚马逊')
```

## 3. 五维正常范围公式

每个维度给出 **正常下限**（P05 拟合）和 **正常中位**（P50 拟合），低于下限则标记异常。

设 `D = avg_devices_per_ip`（设备密度）

### 3.1 应用/IP — 随设备增长（对数关系）

| 情况 | 正常下限 (P05) | 正常中位 (P50) |
|------|:------------:|:------------:|
| D ≤ 10 | **5** | 6~7 |
| D = 30 | **10** | 15 |
| D = 100 | **15** | 22 |
| D = 300 | **30** | 45 |
| D = 1000 | **80** | 250 |

```
app_min = MAX(3, 3 + 4 × ln(D))

  D=5   → 9.4 → 取 5（下限保护）
  D=10  → 12.2 → 取 5（下限保护）
  D=30  → 16.6 → 约 10（实际 P05=8~14）
  D=100 → 21.4 → 约 15（实际 P05=15~28）
  D=300 → 25.8 → 约 30（实际 P05=28~44）
  D=1000 → 30.6 → 约 80（需用分段）

适用范围: 5 ≤ D ≤ 1000
D > 1000: 使用 app / device < 0.05 比率检测
D < 5: 免检
```

### 3.2 型号/IP — 随设备增长（近线性）

| 情况 | 正常下限 (P05) | 正常中位 (P50) |
|------|:------------:|:------------:|
| D ≤ 10 | **6** | 8 |
| D = 30 | **10** | 18 |
| D = 100 | **28** | 50 |
| D = 300 | **90** | 150 |
| D = 1000 | **300** | 650 |

```
model_min ≈ 0.8 × D （约等于设备密度的 80%）

  但这个在低密度区偏高，修正为:
  D ≤ 10:  model_min = 5
  D ≤ 30:  model_min = D × 0.5
  D ≤ 100: model_min = D × 0.6
  D ≤ 1000: model_min = D × 0.65
  D > 1000: model_min = D × 0.3 (NAT 饱和效应)
  
适用范围: D ≥ 5
```

### 3.3 制造商/IP — 增速最慢（有天花板）

| 情况 | 正常下限 (P05) | 正常中位 (P50) |
|------|:------------:|:------------:|
| D ≤ 10 | **3** | 4 |
| D = 30 | **4** | 6 |
| D = 100 | **7** | 8 |
| D = 300 | **10** | 13 |
| D = 1000 | **18** | 27 |

```
mfr_min ≈ 2 + 3.5 × ln(D)
  
  D=5   → 7.6 → 取 3（下限保护）
  D=10  → 10.1 → 取 3（下限保护）
  D=30  → 13.9 → 约 4（实际 P05=4~6）
  D=100 → 18.1 → 约 7（实际 P05=7~10）
  D=1000 → 26.2 → 约 18（实际 P05=18~20）

天花板: 制造商种类有限（全球约 30~60 个主要品牌）
  D > 5000 时 mfr 不再增长，不适用公式
```

### 3.4 DAA/DNA — 最稳定指标（几乎不随密度变化）

| 网络类型 | 正常下限 (P05) | 正常中位 (P50) | 极端异常 |
|----------|:------------:|:------------:|:--------:|
| 移动 | **7** | 11 | < 3 |
| WiFi | **9** | 18 | < 5 |
| 混合 | **5** | 12 | < 3 |

```
DAA/DNA 不需要公式，用分段固定值:
  移动出口: daa_min = 7（移动 P05 全桶最低 7.19）
  WiFi 出口: daa_min = 9（WiFi P05 全桶最低 9.43）
  其他:      daa_min = 5

绝对底线: daa < 3 = 确认刷机造假
```

### 3.5 上报/IP — 随设备线性增长

| 情况 | 正常下限 (P05) | 正常中位 (P50) |
|------|:------------:|:------------:|
| D ≤ 10 | **30** (移动) / **50** (WiFi) | 33 / 87 |
| D = 30 | **50** / **90** | 84 / 160 |
| D = 100 | **120** / **240** | 230 / 460 |
| D = 300 | **350** / **670** | 550 / 1,600 |
| D = 1000 | **900** / **1,200** | 1,050 / 3,000 |

```
移动: report_min ≈ D × 3.0  (上报/设备比 ≈ 3)
WiFi: report_min ≈ D × 5.0  (WiFi 设备上报频率更高)

适用范围: D ≥ 5
D > 5000: 上报量可能达到百万级，不设上限
```

## 4. 综合异常评分体系

### 4.1 五维偏离评分

```
对每个块，计算五个维度是否低于正常下限:

维度1: app/IP < app_min(D)                      → +1 分
维度2: model/IP < model_min(D)                   → +1 分
维度3: mfr/IP < mfr_min(D)                      → +1 分
维度4: DAA/DNA < daa_min(网络类型)               → +1 分
维度5: report/IP < report_min(D, 网络类型)       → +1 分

总分 0-5, 含义:
  0-1: 正常（单维度偏低可能是正常变异）
  2:   可疑（两个维度同时偏低，需关注）
  3+:  高度异常（三维以上协同偏低，造假概率高）
```

### 4.2 为什么多维度交叉有效

**造假的本质困境**:
- 伪造 **设备 ID** → 设备数量膨胀，但 model/device 比率下降
- 伪造 **机型** → model 数量上升，但 manufacturer/model 比率异常
- 伪造 **应用** → app 数量上升，但 DAA/DNA 变化异常
- **任何单一维度的伪造，都会在其他维度产生异常信号**

```
正常网络的维度比例:
  app : model : mfr : daa : report ≈ 协同增长

造假网络:
  某 1-2 个维度膨胀,其他维度不变或反向
  → 维度间比例失调 → 评分 ≥ 2
```

### 4.3 特殊值处理（不参与评分）

| 条件 | 处理 |
|------|------|
| ip_count = 1 | 单独标签（云/VPN/代理出口），不参与比例评分 |
| ip_count ≤ 3 且 dev/IP ≤ 5 | 豁免，样本量不足 |
| 运营商 = 云服务商 | 单独标签，不参与比例评分 |
| dev/IP > 10,000 | 使用 per-device 比率而非 per-IP 值 |
